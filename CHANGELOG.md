# Changelog

## 0.9.7 - 2026-07-17

- 跟随鼠标时支持按 `Esc` 立即取消剩余放置，并自动回滚当前未完成的电容组；
- 移除每组落位前后的两个固定 `250ms` 等待，改为命令完成后立即进入移动、网格和连接校验；
- 多颗电容的引脚读取改为并行执行，并移除每组完成后的重复全页图元扫描；
- README 新增“猫咪快速上手”说明，所有图片继续使用安装包内普通文件路径，不使用 Base64/Data URI。

## 0.9.6 - 2026-07-17

- 修正自动 Release 的 SHA-256 清单，只记录下载后的文件名，可直接运行 `sha256sum -c` 校验；
- 延续 v0.9.5 的整组原生网格吸附与电气顶点校验修复。

## 0.9.5 - 2026-07-17

- 整组落位后调用编辑器原生“对齐网格”，修复以包围盒中心移动时产生的半格偏移；
- 新增电容引脚、标签和全部导线顶点的网格校验，不对齐时自动回滚；
- 主分支 CI 通过后自动创建对应版本的 GitHub Release 并上传 `.eext` 与 SHA-256 校验文件；
- README 图片改用安装包内的普通 PNG 相对路径，兼容不支持 Base64 图片的扩展广场渲染器。

## 0.9.4 - 2026-07-17

- 将主容与去耦器件、封装显示信息和各自容量合并保存为带版本的持久偏好；
- 自动迁移旧版分散的器件与容量设置，插件升级或重新打开后继续沿用用户选择。

## 0.9.3 - 2026-07-17

- 整组落位后自动核对每颗电容的连续位号、真实 Power/GND 标签网络、所有器件与导线的统一位移；
- 母线必须同时解析到目标电源网络和 `GND`，任一自校验失败都会回滚当前组，不再静默报成功。

## 0.9.2 - 2026-07-17

- 修复用户选中的目标芯片仍保留在选择集内，导致整组移动保护误判为“包含 1 个页面既有图元”的问题；
- 选中临时电容组前先清空原有芯片选择，并验证选择集确实为空后再进入原生整组移动工具。
- 修复 API 直接创建的电容保留为 `C?`：生成前扫描当前图页并显式分配连续、不冲突的 `Cxx` 位号；
- 移除落位后修改 Power 标志网络名的路径，直接以真实电源网络创建标签，避免可视文本被客户端重置为默认 `+5V`。

## 0.9.1 - 2026-07-17

- 修复整块移动时真实 `VOUT_PA`/`GND` 网络可能扩展选中页面既有图元，触发“导线不能连接到有非连接标识的引脚上”的问题；
- 临时组使用不可见隔离网络名，落位后再恢复真实电源名与 `GND`；
- 移动前核对实际选中 ID，发现任何页面既有图元即中止并回滚。

## 0.9.0 - 2026-07-17

- 改用 V3.2.149 客户端原生多图元中心移动交互：电容、标签、母线与支路作为完整组随鼠标移动；
- 每个电源域先在画布外构建并校验完整连接，再选中全部图元并执行 `MOVE_BY_CENTER_POINT`，用户一次点击整体落位；
- 整组移动工具就绪后才接受鼠标落位，避免把准备阶段点击误判为放置；
- 不使用剪贴板、不复制图元且保留原位号；客户端不提供多图元移动命令时使用单锚点兼容回退。

## 0.8.1 - 2026-07-16

- 修复逐组放置时首个点击可能被跟随器基线忽略，导致连续落下 C97、C98、C99 等重复锚点的问题；
- 在隐藏配置窗口前完成跟随器基线采集，将落位轮询从 100ms 缩短到 20ms；
- 结束官方放置工具后只保留当前组一个稳定锚点，并自动清理本轮多余锚点。

## 0.8.0 - 2026-07-16

- 将一个锚点自动排完整批改为逐电源域连续放置：每组点击一次决定位置，生成完成后自动进入下一组；
- 每个电源域成为独立增量事务，后续组取消或失败时保留已经成功放置的前序组；
- 鼠标提示实时显示当前组序号和电源标签，全部完成后自动结束并恢复配置窗口；
- 主操作明确显示“逐组放置 N 个电源域”，不再出现“替换现有去耦”。

## 0.7.0 - 2026-07-16

- 改为纯增量追加：每次放置只新增当前计划，不检查、不替换、不删除已有去耦；
- 移除生成记录、撤销整批、删除已生成电容与 Manifest 持久化，缩短生成路径并精简界面；
- 同域独立电容以 3 路受控并发准备；电源域和相交导线保持顺序创建，并发失败会等待已启动任务结束后再统一回滚；
- 电源标签与唯一 GND 标志固定在第一个电容的正上方和正下方，母线从第一个电容向右连接。

## 0.6.3 - 2026-07-16

- 生成、删除电容、删除电源域或撤销批次后立即刷新主操作状态，已有本扩展结果时马上显示“替换现有去耦”，避免用户误以为再次点击会叠加生成。

## 0.6.2 - 2026-07-16

- 为手工注入的原理图右键命令补齐扩展 UUID 命名空间，使客户端能够将“生成去耦”准确分发到本扩展导出函数；
- 已在嘉立创 EDA 专业版 V3.2.149 中验证一级“去耦喵”、二级“生成去耦”均可见，且二级命令能实际打开生成窗口。

## 0.6.1 - 2026-07-16

- 将芯片右键菜单入口改为一级“去耦喵”与二级“生成去耦”，减少原生菜单占用并为后续管理命令预留扩展空间。
- 将右键入口插到原生菜单首个分隔区之后，避免长菜单底部的滚动与点击穿透；
- `draw_end` 后不再依赖临时图元 ID，改用用户落点坐标匹配最近的正式电容，修复“未找到已落下的锚点电容”；
- 底部器件区增加两类全局容量输入，一次覆盖当前全部主电容或引脚去耦电容，并持久化为后续默认值；
- 原生器件选择后优先从 `Value`、`Capacitance`、`容量` 等可信参数同步容量，支持 `104`，同时排除 `0603` 等封装号误判。

## 0.6.0 - 2026-07-16

- 将生成窗口重构为 800 × 540 的单列电源域工作台，同值逐引脚去耦聚合为 `100nF × N`，完整引脚列表移入“调整引脚”抽屉；
- 移除品牌区、四格统计、重复类型说明和常驻高级控件，并将主电容小勾选放到电源域总览行；
- 器件选择合并到底部单行工具栏，删除“当前”备用按钮，主操作成为唯一视觉焦点；
- 为单个芯片的原理图右键菜单增加“去耦喵：生成去耦”入口，并保留顶部菜单作为兼容回退；
- 扩展改为编辑器启动后激活，并以幂等兼容层适配嘉立创 EDA 专业版 V3.2.149 的原理图菜单事件；
- 同一芯片再次生成时默认替换去耦喵自身的旧批次，删除改为逐图元校验，避免反复测试叠加或丢失失败记录；
- 位号与显示容值左移后回读属性坐标，未真正移动时中止并回滚，不再静默报成功；
- 锚点追踪优先使用用户实际落下的图元 ID，减少临时预览图元被误认的风险。

## 0.5.1 - 2026-07-16

- 新建电容的位号与显示容值文字统一向左移动一个原理图网格，兼容原生器件以 `Name = ={Value}` 显示容值的属性结构，器件和导线位置保持不变。

## 0.5.0 - 2026-07-16

- 不再在芯片电源引脚添加网络标签，生成过程不修改芯片本体；
- 以上下引脚跨度最大的电容确定电源/GND 母线位置，较短器件通过上下竖直支路接入；
- 多电源域行距由 100 增至 130，电容横向间距由 35 增至 40；
- 窗口收紧为 1040 × 660，并进一步精简标题、说明文字和生成区布局；
- 主按钮改为“快速生成全部”，一个锚点完成全部已选网络。

## 0.4.0 - 2026-07-16

- “选择器件”按钮现在暂时隐藏去耦喵并打开嘉立创 EDA 底部原生器件库，点中不同器件后自动返回并回填；
- 提供独立的“用当前”按钮，明确采用原生库当前已高亮器件；
- 将生成窗口收紧为 1080 × 720，并重做为紧凑、现代的工具面板布局。

## 0.3.0 - 2026-07-16

- 将逐电源域放置改为全局“添加电容”：人工落下一个锚点后，一次生成全部已选择电源网络；
- 各网络按独立行排列，使用各自电源母线和单个 GND 标志，整批失败统一回滚；
- 每个电源域增加主电容小勾选，可快速排除主电容且不丢失原有容值配置。

## 0.2.2 - 2026-07-16

- 将完整 GND 母线及全部支路一次创建为单个稳定导线图元；
- 删除单个已生成电容时按剩余电容坐标重建公共母线，避免留下悬空支路或误删整网。

## 0.2.1 - 2026-07-16

- 阵列内部导线不再逐段显式写入网络名，改由唯一电源/GND 标志通过真实连接自然传播网络；
- 消除每根电容支路旁重复显示的 `GND` 文本。

## 0.2.0 - 2026-07-16

- 将逐电容 GND 短线和重复 GND 标志改为连续公共 GND 母线，每个电源域只保留一个 GND 标志；
- 使用真实电源/GND 导线形成完整并联连接，并兼容两类器件符号的不同引脚间距；
- 接入嘉立创 EDA 原生底部器件库选择，主电容与引脚去耦电容可分别选择和缓存器件。

## 0.1.15 - 2026-07-16

- 按嘉立创 EDA 原理图坐标系修正竖直电容的上下端判定：较大 Y 为屏幕上方的电源端；
- 电源母线放在电容上方，GND 短线与标识放在电容下方，匹配参考电源页样式。

## 0.1.14 - 2026-07-16

- 在 `draw_end` 后重新对比放置前图元 ID 基线，以差集中的稳定图元作为锚点；
- 兼容客户端在结束重复放置工具时替换临时图元 ID 的行为。

## 0.1.13 - 2026-07-16

- `draw_end` 结束放置工具后，按 primitive ID 重新读取已落下的稳定锚点对象；
- 避免使用工具结束前的失效图元包装对象调用 `modify()`。

## 0.1.12 - 2026-07-16

- 捕获用户点下的锚点后调用客户端 `draw_end` 命令，结束官方接口默认保持的重复放置工具；
- 只保留已落下的锚点，然后再执行旋转、容值写入和连接式阵列生成。

## 0.1.11 - 2026-07-15

- 在电容修改、其余器件创建和真实导线创建完成前保持画布命令上下文，避免过早恢复 iframe 导致 `modify()` 抛出 `cmdKey` 异常。

## 0.1.10 - 2026-07-15

- 在恢复扩展窗口前先清理鼠标跟随提示；
- 将已脱离画布上下文时可能抛出 `cmdKey` 的提示清理操作变为幂等操作，不中断电路生成事务。

## 0.1.9 - 2026-07-15

- 同步为父编辑器窗口提供经 iframe 偏移换算的 `clientX/clientY`，兼容官方鼠标放置方法在落点阶段读取父窗口 `window.event` 的实现。

## 0.1.8 - 2026-07-15

- 将 iframe 内的可信点击事件临时桥接到官方放置接口依赖的旧式 `window.event`，解决 Electron 中 `clientX` 解构异常；
- 官方放置调用启动后立即恢复全局事件属性，不污染后续 UI 交互。

## 0.1.7 - 2026-07-15

- 在用户点击处理器的同步阶段立即调用 `placeComponentWithMouse()`，保留 V3.2.149 依赖的真实 `window.event` 和鼠标坐标；
- 预先缓存图元 ID 基线，避免在启动放置工具前异步等待。

## 0.1.6 - 2026-07-15

- 隔离 V3.2.149 在已成功恢复 iframe 后仍抛出的内部 `cmdKey` 异常，不让它中断后续连接式电容阵列生成。

## 0.1.5 - 2026-07-15

- 落点捕获后使用扩展窗口重获焦点，避免 V3.2.149 在放置工具中调用 `activateDocument()` 时抛出 `cmdKey` 异常。

## 0.1.4 - 2026-07-15

- 区分官方接口的临时鼠标图元与用户真正点下的锚点图元；
- 改用新增图元 ID 轮询捕获落点，兼容 V3.2.149 未回调 `SCH_Event.create` 的情况。

## 0.1.3 - 2026-07-15

- 改为按电源域逐个调用官方 `placeComponentWithMouse()` 人工落位；
- 电容阵列使用水平电源母线和逐电容 GND 短导线真实连接；
- 删除自动散放以及每个电容重复放置电源标签的布局；
- Manifest 分开记录器件与导线，支持精确回滚和删除。

## 0.1.2 - 2026-07-15

- 修复 iframe 相对资源路径和 ES module 脚本在客户端未加载的问题；
- 扩展 VIN_SRAM、VDDIN、VIOIN_18 等实际雷达芯片电源脚识别；
- 补充“显示在顶部菜单”的安装配置说明。

## 0.1.1 - 2026-07-15

- 修复器件尚未进入网表时生成窗口无法打开的问题；
- 增加编译后入口运行时回归测试，覆盖无网表器件场景；
- 明确网表只用于补充标签建议，不再成为生成器启动条件。

## 0.1.0 - 2026-07-15

- 建立“去耦喵”嘉立创 EDA V3 扩展；
- 支持选中一个芯片并读取全部引脚及 `.enet` 网络名；
- 支持 POWER 类型、常见名称和已有网络名的多电源域建议；
- 支持手动选择任意引脚、新建/删除电源域和重新分配引脚；
- 支持预设或自定义电源标签；
- 支持每域任意增删主电容和逐引脚去耦电容；
- 支持搜索并缓存用户选择的两引脚电容库器件；
- 支持事务式创建与失败回滚；
- 支持删除单个电容、删除电源域和撤销整批生成；
- 增加 Apache-2.0 许可证、测试和 CI。
