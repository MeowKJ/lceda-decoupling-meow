# 去耦喵架构设计

## 产品模型

```text
选中芯片
└─ 多个电源域
   ├─ 可编辑电源标签
   ├─ 0~N 个电源引脚
   ├─ 0~N 个主电容
   └─ 每个引脚 0~N 个去耦电容
```

自动识别只创建初始计划。引脚、电源域、标签和电容配置均可由用户覆盖。

## 数据流

```text
SCH_SelectControl.getAllSelectedPrimitives()
  + SCH_PrimitiveComponent.getAllPinsByPrimitiveId()
  + SCH_ManufactureData.getNetlistFile()
                         │
                         ▼
               GeneratorInput v1
                         │
                         ▼
         POWER 类型 + 名称 + 现有网络名建议
                         │
                         ▼
             用户编辑多电源域生成计划
                         │
                         ▼
 sys_PanelControl.openBottomPanel(LIBRARY)
  + lib_SelectControl 读取原生库选择
                         │
                         ▼
  createNetFlag() + PrimitiveComponent.create()
                         │
                         ▼
              增量追加 / 本批失败回滚
```

## 初始识别

候选电源引脚满足任一条件：

1. 引脚类型为 `ESCH_PrimitivePinType.POWER`；
2. 名称匹配 VDD、VCC、VDDA、AVDD、DVDD、VBAT、VREF、VCORE等模式。

GND、VSS、AGND等地引脚会从候选中排除。

初始电源标签按以下优先级生成：

1. 引脚已经连接到明确的非自动网络名；
2. 否则严格保留原始引脚名，例如 `VDD` → `VDD`、`VDD33` → `VDD33`、`VDD_3V3` → `VDD_3V3`；
3. 不根据名称换算电压，也不把 `VDD33` 自动改写为 `+3V3`；
4. 引脚名为空时使用 `VDD`，等待用户修改。

相同建议标签的候选引脚初始合并为同一电源域。用户可以新增域并重新分配引脚，从而完成拆分或合并。

网表只用于补充现有网络名建议：读取失败、器件未进入网表或尚未分配位号时，仍然使用符号引脚信息继续生成。

## 默认配置

- 每个电源域：`4.7uF × 1` 主电容；
- 每个电源引脚：`100nF × 1` 去耦电容；
- 接地标签：`GND`；
- 两类全局容量：底部输入框直接覆盖当前全部同类电容，并分别持久化为后续新电源域的默认值；
- 电容库器件：点击底部“主容器件”或“去耦器件”时，通过 `sys_PanelControl.openBottomPanel(ESYS_BottomPanelTab.LIBRARY)` 打开嘉立创 EDA 底部原生库，并临时隐藏生成窗口；随后轮询 `lib_SelectControl.getSelectedLibraryRowInfo()`，检测到用户点中不同器件后自动回填并恢复窗口，分别缓存两类器件的 `{ libraryUuid, uuid }`。若 `lib_Device.get()` 返回的器件数据含有明确容值参数，还会同步对应的全局容量。

官方 API 没有提供“原生库选择完成”的回调，因此界面通过轮询检测选择变化；超时后恢复窗口并提示用户重新选择。

## 鼠标落位与创建事务

所有已选择电源域按顺序作为独立整块移动事务放置：

1. 在画布外坐标使用不可见隔离网络名构建当前电源域的完整电容、标签、母线和支路，并完成容值、文字位置与连接校验；
2. 读取客户端合并后的稳定导线 ID，先清除用户原先选中的芯片，再使用 `sch_SelectControl.doSelectPrimitives()` 选中整组，并核对选中集合没有包含页面既有图元；
3. 调用 V3.2.149 客户端内部 `MOVE_BY_CENTER_POINT` 命令，将选中图元交给原生多图元移动工具并以整组中心为鼠标参考点；
4. 隐藏配置窗口并接受用户鼠标落位；一次鼠标抬起由客户端原生移动动作整体提交；
5. 确认首颗电容已经离开画布外临时坐标，将两个网络标识恢复为真实电源名与 `GND`，随后刷新图元基线并准备下一组；
6. 每域主电容排在左侧，逐引脚电容向右排列并写入 `Value`；取消主电容勾选时跳过该域主电容；
7. 位号与实际显示容值属性左移一个网格并回读坐标；若 Beta 属性接口没有应用修改，则只回滚当前组；
8. 同一电源域最多并发准备 3 颗互不相连的电容；每个电源域按顺序完成，同一电容的位号和容值文字并行移动；
9. 找出上下引脚跨度最大的电容，以其上端和下端分别确定电源/GND 母线位置；电源标签和单个 GND 标识与第一个电容保持同 X 坐标；
10. 两条母线从第一个电容向右延伸，较短电容的上下引脚分别通过按顺序创建的竖直支路接入公共母线，避免相交导线的拓扑竞态；
11. 不在芯片电源引脚添加标签，不修改芯片本体，也不读取或改写已有去耦；
12. 每个成功组立即成为独立增量结果；后续组取消或失败时保留前序组，生成完成后不保存图元 Manifest。

如果任一步失败，并发调度会先等待已启动任务结束，再使用已收集 ID 与放置前导线基线回滚本次创建内容；删除接口返回失败时会明确提示用户检查本次新增图元。

## 布局策略

- 用户为每个电源域单独确定锚点，组与组之间的位置和间距完全由用户控制；
- 主电容排在阵列左侧，逐引脚去耦电容依次向右排列；
- 电容统一竖直放置，最长电容的上下端确定两条水平母线的位置；
- 较短电容通过上下竖直支路接入母线，电源标签和唯一 GND 标识位于第一个电容上下方；
- 所有坐标使用原理图单位 `0.01 inch`。

嘉立创 EDA 原理图坐标的较大 Y 值对应屏幕上方；因此电源端选择较大 Y 引脚，GND 端选择较小 Y 引脚。

公共 API 只能让单个库器件随鼠标。完整组预览使用 V3.2.149 已存在的客户端内部 `MOVE_BY_CENTER_POINT` 多图元移动动作，并保留单器件锚点模式作为兼容回退。

## 安全边界

- 扩展没有网络服务或遥测；
- 不读取或改写已有去耦电容；
- 不移动用户图元；
- 创建接口失败时尽力回滚；
- 生成事务期间不支持协作者或其他扩展同时写入导线；
- 用户仍需检查符号方向、原理图排版和 PCB 物理去耦位置。
