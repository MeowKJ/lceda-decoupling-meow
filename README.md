# 原理图自动去耦喵（lceda-decoupling-meow）

![原理图自动去耦喵图标](./images/logo.png)

[![CI](https://github.com/MeowKJ/lceda-decoupling-meow/actions/workflows/ci.yml/badge.svg)](https://github.com/MeowKJ/lceda-decoupling-meow/actions/workflows/ci.yml)
[![License](https://img.shields.io/badge/license-Apache--2.0-blue.svg)](./LICENSE)

原理图自动去耦喵（简称“去耦喵”）是一个面向嘉立创 EDA 专业版 V3 的原理图生成扩展：选中一个芯片后，它会识别多个候选电源域，让用户确认引脚和电源标签，再批量添加主电容与逐引脚去耦电容。

自动识别只是建议。用户可以选择任意引脚、新建或删除电源域、拆分或合并引脚归属，并为每个电源域单独增删电容。

![原理图自动去耦喵在嘉立创 EDA 中生成两组独立去耦网络](./images/generated-groups.png)

真实编辑器结果：每个电源域独立整块落位，共享连续电源/GND 母线。

![原理图自动去耦喵操作演示](https://cdn.jsdelivr.net/gh/MeowKJ/lceda-decoupling-meow@v0.9.15/images/usage-demo.gif)

## 使用说明 ฅ^•ﻌ•^ฅ

1. **启动扩展**：在原理图画布中右键目标芯片，选择“原理图自动去耦喵 → 生成去耦”；
2. **核对电源域**：检查自动识别的电源网络；如需调整，可使用“调整引脚”增删引脚或修改分组；
3. **配置器件与容量**：分别选择主电容和引脚去耦器件，并确认相应容量；扩展会自动保存这些使用偏好；
4. **确认主电容配置**：对于不需要主电容的网络，可直接取消电源域总览行中的勾选；
5. **依次完成放置**：点击“整块放置”，按照画布提示移动并单击，每次放置一个完整的电容母线组；
6. **取消放置操作**：跟随鼠标期间按 `Esc`，扩展会终止剩余放置，并清理当前尚未完成的电容组。

> 使用提示（=^･ω･^=）：去耦喵仅负责生成原理图连接；进行 PCB 布局时，仍应将去耦电容尽量靠近芯片电源引脚。

## 功能

- 识别符号类型为 `POWER` 的引脚；
- 补充识别 VDD、VCC、VDDA、VBAT、VREF、AVDD、DVDD、IOVDD、VCORE等常见名称；
- 一个芯片支持多个独立电源域；
- 初始菜单以明确网络名分组：引脚名称不同但网络名相同仍属于同一电源域，网络名不同则分别显示；
- 电源域总览显示网络名称和具体引脚号，展开后显示每个引脚的名称、引脚号及去耦配置；
- 所有芯片引脚都可手动纳入或移出电源域；
- 每个电源域可选择常用标签或输入自定义标签；
- 默认每个电源域添加一个 `4.7uF` 主电容；
- 默认每个电源引脚添加一个 `100nF` 去耦电容；
- 底部器件区可全局修改主电容和引脚去耦容量，立即同步覆盖当前全部电源域；
- 主电容与逐引脚电容均支持任意添加、删除和修改值；
- 每个电源域可用小勾选快速排除主电容，并保留原有主电容配置；
- 默认界面将同值去耦聚合为 `100nF × N`，只有展开电源域时才显示逐引脚例外；
- 点击一次“整块放置 N 个电源域”后，插件按电源域依次将完整电容组黏到鼠标；
- 鼠标预览包含电容、标签、母线和支路；用户每点击一次整体落下一组，然后自动进入下一组；
- 不在芯片电源引脚添加标签，不修改芯片本体；
- 以上下引脚跨度最大的电容为基准生成电源/GND 母线，较短器件使用竖直支路接入；
- 电源标签与唯一 GND 标志分别位于第一个电容正上方和正下方；
- 每个电源域独立落位；同组电容受控并发准备，相交导线按顺序稳定连接；
- 点击底部“主容器件”或“去耦器件”会暂时隐藏去耦喵并打开原生器件库，点中器件后自动返回并回填；若原生器件带有 `Value`、`Capacitance`、`容量` 等参数，还会同步对应的全局容量；
- 主容和去耦的器件、封装显示信息与各自全局容量会持久保存；升级后自动迁移旧偏好，下次打开直接沿用；
- 生成过程失败时自动回滚本次已创建图元；
- 每次生成只追加当前计划，不检查、不替换、不删除已有去耦；
- 新建电容的位号与显示容值向右移动一个网格后会回读坐标校验，移动失败则回滚本次生成；
- 在画布中右键单个芯片，可直接选择“原理图自动去耦喵 → 生成去耦”；顶部菜单入口仍然保留。

## 标签建议原则

去耦喵不会根据引脚名换算或猜测电压标签：

- 能读取到明确现有网络名时，优先采用该网络名作为建议；网表不可用时仍可继续生成；
- 没有明确现有网络名时，严格沿用芯片引脚名称，例如 `VDD` → `VDD`、`VDD33` → `VDD33`、`VDD_3V3` → `VDD_3V3`；
- 不会将 `VDD33` 或 `VDD_3V3` 自动改写为 `+3V3`；
- 用户始终可以覆盖自动建议。

## 安装与使用

### 从源码构建

需要 Node.js 20.17 或更新版本。

```bash
npm install
npm run check
```

生成的扩展包位于：

```text
build/dist/lceda-decoupling-meow_v0.9.15.eext
```

在嘉立创 EDA 专业版 V3 中进入“高级 → 扩展管理器 → 导入”，选择 `.eext` 文件。导入后切换到“已安装”，选择“原理图自动去耦喵 → 配置”，勾选“显示在顶部菜单”；否则入口只会收纳在“高级 → 原理图自动去耦喵”子菜单中。

### 使用

1. 打开一个原理图图页；
2. 在画布中右键一个芯片，选择“原理图自动去耦喵 → 生成去耦”；也可以先只选中一个芯片，再使用顶部“原理图自动去耦喵 → 为选中芯片生成去耦”；
3. 在电源域总览中确认自动识别结果；需要调整时点击“调整引脚”，搜索、勾选并重新分配；
4. 点击某个电源域可展开高级编辑，修改标签、增加电容或设置逐引脚例外；
5. 不需要主电容的网络可直接在总览行取消小勾选，无需展开；
6. 点击底部“主容器件”或“去耦器件”，窗口会暂时隐藏并打开原生器件库；点中器件后自动返回并回填。旁边的容量输入框会全局修改这一类电容；器件参数中存在可靠容值时会自动同步；
7. 点击“整块放置 N 个电源域”；
8. 根据鼠标提示依次移动并点击每个完整电容组；每次点击整体落位，然后自动切换到下一组；任何时候按 `Esc` 都会取消剩余放置并清理当前未完成的组。

## 生成方式

- 不在芯片电源引脚创建标签；
- 用户为每个电源域直接移动并点击一个完整图元组，像原生复制粘贴一样安排位置；
- 主电容和逐引脚去耦电容竖直并排，最长电容的上端和下端分别确定两条母线位置；
- 较短电容通过上下竖直支路接入连续母线，电源标签和唯一 `GND` 标志与第一个电容同 X 坐标；
- 不移动芯片，不检查或删除用户已有器件；
- 重复运行会在新锚点追加一组，不维护或修改旧批次。

## 当前限制

- 只处理一个选中芯片；
- 文字右移只对本次新建电容生效，已有电容不会被自动改写；
- 芯片右键入口使用了对 V3.2.149 原理图菜单事件的保护性兼容层；若后续客户端内部菜单结构变化，可继续使用顶部菜单入口；
- 原生库参数同步只读取明确的 `Value`、`Capacitance`、`容量` 等字段或带单位的名称/描述，不会把 `0603` 等封装号猜成容量；器件没有可靠参数时保留用户当前输入；
- 电容值只作为器件 `Value` 写入，不判断容值、耐压和介质是否合理；
- 完整组预览使用 V3.2.149 客户端内部 `MOVE_BY_CENTER_POINT` 多图元移动命令；不使用剪贴板，也不会复制图元或覆盖用户复制内容；
- 若未来客户端移除该命令，会自动回退为单锚点逐组放置；
- 生成位置不代表 PCB 去耦电容的最终物理位置；
- 只生成阵列内部的连续上下母线，不跨越原理图自动拉长导线；
- 生成期间不要让协作者或其他扩展同时新增导线；失败回滚需要用放置前导线基线识别客户端合并后的本批线段；
- 创建接口在官方文档中仍标记为 Beta，建议先在副本工程验证。
